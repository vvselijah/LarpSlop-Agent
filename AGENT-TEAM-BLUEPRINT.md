*Produced 2026-06-12 by a 104-agent deep-research run: 22 sources fetched, 105 claims
extracted, top 25 adversarially verified (3 independent votes each) → 15 confirmed,
10 killed. Re-verify pricing/feature specifics at install time — this ecosystem moves
fast enough that two claims changed during the research window itself.*

## The one-paragraph verdict

You already own the hard parts: working Meta Ads + Instagram MCPs, a self-feeding
analytics dashboard, a pro-grade video editing/generation suite, and a structured
vault. The evidence says the winning pattern at your scale is **plain editable
markdown context + a few first-party MCPs**, not plugin packs or "AI social media
manager" suites. Exactly one paid tool is worth adding now (Metricool, to unlock
YouTube/TikTok). Everything else is either already built (as of today), gated on a
real trigger, or a documented gimmick.

## Tier A — add now (one item)

### Metricool MCP — the YouTube/TikTok unlock
- **What:** Official first-party MCP; scheduling + analytics across IG, TikTok, YouTube,
  Facebook, X, LinkedIn, Pinterest, Threads, Bluesky, Twitch, plus Meta/Google/TikTok ads
  data. ~27 tools (`post_schedule_post`, `get_tiktok_videos`, `get_youtube_videos`, …).
  Explicitly supports Claude Code. (3-0 verified against vendor docs.)
- **Why it's the unlock:** TikTok restricts unaudited API clients to **private-only posts**
  (3-0, TikTok's own docs) — you cannot build your own TikTok publisher. Metricool is an
  audited client, so scheduling through it sidesteps the wall. It also covers YouTube,
  where no claim about direct-API quotas survived verification.
- **Setup (needs Elijah, ~30 min):** create Metricool account → connect IG/TikTok/YouTube
  via OAuth inside Metricool → follow https://metricool.com/metricool-mcp-claude/ to add
  the MCP (API token from Metricool settings). Works on any plan; real use likely needs
  paid (~$22–49/mo). The "requires Advanced tier" claim floating around was **refuted**.
- **Consolidates:** cross-platform scheduling, TikTok/YouTube analytics, competitor tracking.
- **Open-source alternative:** Postiz (official MCP, 30+ platforms, write-only on posts).
  Prefer Postiz **cloud** — self-hosted has documented MCP transport issues.

## Tier C — skip (researched gimmicks & traps)

- **Building your own TikTok publisher MCP** — unaudited = private-only posts + 5 users/day cap. Dead end by TikTok policy (3-0). 
- **Marketplace plugin packs / multi-step packaged content automations** — documented abandonment pattern: too rigid, token burn, "fighting the format." Your editable files + skills do the same job and stay correctable (3-0).
- **Unvetted community MCP servers** — the directory hit ~18,000 servers (live-verified); listing ≠ quality. Default to first-party (Meta, Whop, beehiiv, Metricool, Zapier, Postiz all qualify).
- **Migrating the hub to Cowork** — for batch/file-heavy ops work, Claude Code desktop was ~4–10× faster in practitioner tests (VM overhead, per-turn tool caps). Keep Cowork for what it's good at; the hub stays here (3-0, anecdotal).
- **Auto-DM closers / "AI social media manager" suites** — no non-anecdotal evidence survived that they move revenue at 100k-follower scale, and Meta automation-policy risk is real. Your `instagram` MCP already reads/sends DMs for human-in-the-loop workflows; full automation is unproven. (Open question — revisit if evidence appears.)

## Hard constraints to design around (all 3-0 verified on primary docs)

1. **IG API publishing:** 100 posts/24h (plenty), **but** API-published reels can't use
   licensed/trending audio, images are JPEG-only, no shopping tags. → Policy: API-publish
   voice-only/original-audio content; trend-audio reels stay manual or via Metricool.
2. **TikTok:** see above — audited schedulers only.
3. **Don't trust these refuted "facts":** "IG caps at 50 posts/day" (it's 100), "Metricool
   needs Advanced tier" (any plan), "major platforms have no official MCPs" (false),
   "$355/mo is the realistic cost benchmark" (refuted; actual cost depends on usage).

## Watch-outs (flagged but UNVERIFIED — treat as risks, not facts)

- **OneDrive + high-churn repos:** this workspace lives in OneDrive. Fine for docs/configs;
  for future code-heavy repos with git + agent workloads, prefer a non-synced path like
  `C:\dev\` and back up separately. (The whole Windows/OneDrive research angle produced
  no verified claims — nobody's measured it properly.)
- **Setup-time reality check:** one practitioner's full business-OS build took 40–60 hours
  over two months (self-reported). Mitigation: this blueprint's trigger-gated approach —
  never build ahead of need.
