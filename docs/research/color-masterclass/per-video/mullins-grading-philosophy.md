# mullins-grading-philosophy — analysis

*Source: Mitchell Mullins, 137K views · framesRead: 28*

## Overview
A philosophy-first DaVinci Resolve masterclass built around one organizing idea: every adjustment falls into one of two "buckets" — CORRECTIONS (exposure/balance/contrast — things that "could have been done in camera," done photometrically to preserve how the camera saw the world) and LOOK DEVELOPMENT (saturation, hue/channel manipulation, split-tone, film-emulation LUTs — pushing the image past where it naturally lived to make it FEEL a certain way). The technical spine is a node "sandwich": an input Color Space Transform (camera log/gamut → DaVinci Wide Gamut/Intermediate), all grading nodes living inside DWG (the widest, most malleable playground), then an output CST (DWG → Rec.709 Gamma 2.2). Mullins demonstrates a full ~12-node tree (DWG, HDR Exp, Lin Bal, Con, Sat, Matrix, Split Tone, HSV Hue, Post Con, HK, Texture, REC709), but his repeated thesis is "do the most with the least" — most of his real-world grading is just a few correction nodes underneath a film-emulation LUT that does the split-toning and the transform. He uses RED footage (RWG/Log3G10), a Cullen Kelly macOS color-viewer LUT for accurate monitoring, and a set of his own DWG film-emulation LUTs (Terra = his Kodak 2383 emulation). Demonstrated on 4 clips end-to-end (hero portrait, locker-room, widescreen, ARRIRAW open-gate).

## Order of operations
1. 0. PROJECT SETUP first: Color Management = DaVinci YRGB, Timeline = DaVinci WG/Intermediate, Output = Rec.709 Gamma 2.2; set 3D LUT interpolation to TETRAHEDRAL (not Trilinear); load Cullen Kelly macOS color-viewer LUT for accurate monitoring.
2. 1. INPUT TRANSFORM: first node = Color Space Transform from camera (e.g. REDWideGamutRGB / Log3G10) INTO DaVinci Wide Gamut / Intermediate.
3. 2. Build the 'sandwich': matching OUTPUT CST as the LAST node (DWG/Intermediate → Rec.709 Gamma 2.2, Tone Mapping DaVinci, Gamut Mapping ON). Everything between lives in DWG — the widest playground.
4. 3. CORRECTIONS (could-have-been-done-in-camera), broadest brush first: (a) HDR EXPOSURE via the HDR Global wheel — set where the image 'lives' (never Offset). (b) LINEAR BALANCE — gamma set to LINEAR, balance with the GAIN color wheel only (never Temp/Tint). (c) CONTRAST — judge with saturation at zero; set Pivot to middle-gray (DWG ≈ 0.336), add contrast, steer the eye with pivot.
5. 4. LOOK DEVELOPMENT begins at SATURATION: use the Color Slice Saturation (deepest/most natural), not the primaries multiplier or Color Boost/vibrance.
6. 5. MATRIX (3x3 DCTL) for skin-tone channel correction (free DCTL beats the RGB Mixer's manual rebalancing).
7. 6. SPLIT TONE — taught (curves, Lum Mix 0, mono-middle-gray anchor) but in practice REPLACED by a film-emulation LUT; LUT does the warm-highlight/cool-shadow work and the look.
8. 7. Work correction nodes UNDERNEATH the LUT to restore low-end/contrast the LUT washed out.
9. 8. HSV DCTL for granular post-look hue/sat/value (keep skin dynamic).
10. 9. POST CON: a second contrast/log-wheel node to fine-tune blacks and low-end tint after the look.
11. 10. HK (Helmholtz-Kohlrausch) DCTL on everything — subtle perceptual luminance correction / 'icing.'
12. 11. TEXTURE last: sparing film grain (drop grain in highlights/mids, it lives in shadows) and/or Contrast Pop for sharpness, or a slight blur for a pro-mist/black-satin softness.
13. 12. Iterate fast on real jobs by collapsing to a FEW correction nodes + a baked LUT — 'do the most with the least.'

## Node tree
Core structure is a serial "sandwich." Input CST node (labeled "DWG", node 01): Input Color Space = REDWideGamutRGB, Input Gamma = RED Log3G10, Output = DaVinci Wide Gamut / DaVinci Intermediate (f005). Output CST node (labeled "REC709", last node): DaVinci Wide Gamut/Intermediate → Rec.709 / Gamma 2.2, Tone Mapping = DaVinci, Gamut Mapping ON (f007). EVERY node between the two CSTs operates in DWG/Intermediate. The full demonstrated 12-node serial tree (f021/f023/f039, two visual rows): 01 DWG (input CST) → 02 HDR Exp (HDR Global wheel) → 03 Lin Bal (linear-gamma white balance) → 04 Con (contrast+pivot) → 05 Sat (Color Slice saturation) → 06 Matrix (3x3 channel-mixer DCTL) → 07 Split Tone (curves, usually deleted/replaced by a LUT) → 08 HSV Hue (TetraInterpHSV DCTL) → 09 Post Con (log-wheel black/finishing tweaks) → 10 HK (Helmholtz-Kohlrausch DCTL) → 11 Texture (Blur/Sharpen or Contrast Pop + film grain) → 12 REC709 (output CST). He repeatedly says this is "about as many nodes as I'd ever have" and that half could be dropped; the simplified trees (f047 widescreen = ~4 nodes HDR Exp/Lin Bal/Con + baked-in LUT; f043 locker-room) prove the point. When a film-emulation LUT is used, the LUT replaces the Split Tone node (the node is even renamed "LUT").

## Scopes
- **Waveform / RGB Parade** — Reads it as the shape of the signal — uniform lift (bad, Offset) vs anchored bend (good, HDR Global); on the grayscale ramp the waveform is a clean diagonal line (f012). · targets: Primary judgment tool throughout — used to see where the image 'lives,' verify black-point anchoring under HDR vs Offset (data falling off the top = clipping), and watch contrast pivot push energy to highlights or shadows. No fixed broadcast numbers preached; he grades to feel but watches that nothing clips off the cliff. Visible as R/G/B parade bottom-right in nearly every UI frame (f005-f049).
- **Custom Curve / 1D response (grayscale ramp)** — Watches the diagonal bend into an S (contrast) or lift at the toe (LUT washing blacks). · targets: Used to visualize tone response and split-tone moves; with a grayscale ramp the scope shows the literal transfer curve so you can SEE HDR-wheel bending and Contrast-Pop's S-curve (f034 S-shape, f047).
- **3D Color Cube (Nobe OmniScope)** — Visual — sees the cloud expand when saturation is pushed and compress at the gamut walls; on the grayscale ramp it's a single diagonal line of neutrals (f039 area). · targets: A 3D point-cloud of all scene colors in the gamut — used to watch saturation expand/compress and gamut-mapping fold edge colors back into Rec.709 (f027, f035 area, f037).
- **CIE Chromaticity plot** — Watches the color blob stay inside the Rec.709 triangle and stay smooth (no spikes/banding). · targets: Used to watch colors relative to the Rec.709 triangle and spot banding/over-push (Hue-vs-Hue 'blasting from center'); visible in f027 and f037.
- **Histogram** — Used to find where the bulk of the image sits and place the pivot/center point. · targets: Appears with curves (f009/f029) to show tonal distribution while anchoring middle-gray.

## Primaries
CORRECTIONS, in three dedicated nodes, judged broad-to-fine. (1) HDR EXPOSURE node — exposure is set with the HDR palette's GLOBAL wheel (Dark/Shadow/Light/Global wheels, f011), NOT the Offset wheel. Demonstrated on a grayscale ramp (f012): pushing Offset lifts the whole signal uniformly (blacks lift, data clips off the top — "stuff falling off the cliff"), whereas the HDR Global wheel BENDS the curve, anchoring the black point and giving a soft rolloff into highlights — far more retained detail. He treats exposure as "the broadest brush stroke." (2) LINEAR BALANCE node — white balance done the least-destructive way (credited to Cullen Kelly): set the node's GAMMA to LINEAR (under gamma controls), then in the PRIMARIES (color wheels, NOT log wheels) touch ONLY the GAIN wheel to balance. On screen f015 the Gain reads 1.00 / 0.98 / 1.00 / 1.03 (R/G/B/Y) — tiny moves. He contrasts this with the Temp/Tint slider, which compresses/destroys color data by tinting. Grayscale ramp warms cleanly via gain (f034). (3) CONTRAST node — uses Contrast + PIVOT. Key trick: judge contrast with SATURATION PULLED TO ZERO (desaturate the node, f017/f018) because color deceives the eye. Demonstrated Pivot/middle-gray: cranking contrast reveals the pivot point; he locates true middle-gray with a mono middle-gray DCTL and types Pivot = 0.336 as the DWG center pivot (f017 shows Pivot 0.336; the default Con node sits at Pivot 0.435, f019). On-screen contrast values seen: 1.318 (f018), and contrast achieved alternatively via Gain up + Lift down (he notes Contrast = raising Gain & lowering Lift are the same function; f043 Gain 1.00/0.77/1.02/1.44). Pivot can be intentionally pushed into the sky or shadows to steer the eye. "Contrast is probably the most important [tool] at getting an image to look how you want."

## Secondaries
Mullins is anti-mask/anti-qualifier for his core workflow — almost everything is full-frame/primary. SATURATION (f019, f049): he compares Resolve's saturation engines and picks the COLOR SLICE tool's Saturation as most preferable (deeper, more natural, "richer" than the primaries Sat multiplier). Distinctions he teaches: primaries Saturation = a MULTIPLIER (amplifies already-saturated areas, blows them out); Color Boost = ADDITIVE / vibrance (invents saturation in flat areas, breaks shadows apart); HSV/color-warper = harsher; Color Slice Sat = subtractive-feeling, deep. On f049 Color Slice Saturation reads 1.16. MATRIX node (f025/f029): a free 3x3 channel-mixer DCTL with Tetra interpolation, used ALMOST EXCLUSIVELY for skin-tone correction — preferred over the built-in RGB Mixer because the RGB Mixer forces you to manually rebalance every channel (e.g. +12 here forces -2 there), while the DCTL lets you adjust freely. On-screen Matrix DCTL values (f025): Red-Red 1.264, Red-Green -0.016, Red-Blue 0.233, Yellow-Blue -0.078. HSV node (f037/f045): a TetraInterpHSV DCTL giving independent Hue/Sat/Value per color (Red Hue/Sat/Value, etc.) — used post-look for granular, "blood-under-the-skin" skin dynamics; on f045 Red Sat 0.333, Red Value -0.302. He explicitly avoids Hue-vs-Hue curves (banding) and the color warper (harsh) for these moves. No power windows or tracking demonstrated.

## Skin tone
Skin is handled primarily through the MATRIX (3x3) node and the HSV DCTL rather than a vectorscope skin-line qualifier. His stated philosophy: keep skin DYNAMIC and SEPARATED, not flattened to one magenta hue — he likes to "see the magenta and the shift to yellow… like you can see the blood underneath the skin." In the Matrix node he opens magenta into orange skin by adjusting Red/Blue (f033 area: "the skin looks a little orange, so I'll come in red and blue… open up the magentas"). He references checking the vectorscope/CIE plot (the CIE chromaticity scope is visible in f027/f037) but does not teach a numeric skin-line angle target; the method is by-eye in DWG using the channel matrix and HSV, with split-toning/LUT doing the larger warm-highlight/cool-shadow skin relationship. No explicit vectorscope-skin-line "11 o'clock / I-bar" instruction was given numerically.

## Color management
DaVinci YRGB (project-level color management; NOT ACES, NOT full DaVinci YRGB Color Managed/DRCM — he uses explicit Color Space Transform nodes rather than auto-managed). Seen on screen in Project Settings → Color Management (f003, f013): Color science = DaVinci YRGB; "Use separate color space and gamma" checked; Timeline color space = DaVinci WG/Intermediate (DWG/Intermediate); Output color space = Rec.709 Gamma 2.2. Deliberate choice of gamma 2.2 over the traditional 2.4: he argues for grading under how the work will actually be VIEWED (MacBook/iPhone/YouTube are optimized for 2.2); 2.4 remains correct for cinema/broadcast deliverables. Dolby Vision / HDR10+ / HDR Vivid all left disabled. Lookup Tables panel (f025): 3D lookup table interpolation switched from the default Trilinear to TETRAHEDRAL — he stresses this hard ("I cannot stress this enough"); leaving it Trilinear causes banding/compression when LUTs are applied. Color viewer lookup table = "CK macOS Viewing Transform v1.3" (Cullen Kelly) — a monitor-only viewing transform that is NOT baked into the render, used to grade accurately under the final display. Input/Output/Video-monitor LUTs = None. Scopes lookup table = "Use video monitoring selection." Tone-mapping method inside the CST set to DaVinci; Gamut Mapping in the output CST turned ON (saturation compression) so out-of-Rec.709 colors fold back in gracefully.

## Look design
LOOK DEVELOPMENT is where "feel" comes from. SPLIT-TONING theory (f029/f031): all split-toning happens in the CUSTOM CURVES, with the curve's LUMINANCE MIX set to 0 (critical, "don't ask why, just do it") so RGB channels move independently of luma. Before pulling curves he drops a free "Mono middle-gray" DCTL (by "Mono Nodes") to find the true R/G/B center point in DWG, pokes the curve there to anchor it, then raises reds in highlights / lowers reds in shadows etc. to create COLOR CONTRAST (warm highlights, cooler/magenta shadows — the classic teal-orange instinct: brains find warm-highlight/cool-shadow more inviting). BUT his big reveal: he personally does NOT hand-split-tone — he lets LUTs do it. FILM EMULATION: he compares popular free Kodak 2383 print-emulation LUTs and notes their flaw — they lift/fade the blacks (washed low end) realistically but undesirably. He built his own DWG LUT pack (f035: Terra, Tetra, Pyro, Nova, Helios, Ember, Dusk, Bloom, Aether — all "_DWG"). TERRA is his hand-tuned Kodak 2383 emulation adapted to NOT crush/lift the low end. Workflow: drop the LUT (it replaces Split Tone), then work the correction nodes UNDERNEATH the LUT (raise low end, adjust contrast) so the film split-tone is retained while keeping shadow detail. He stacks/cycles LUTs freely ("slap two LUTs on, looks good, ship it") and dials opacity via the node Key output (e.g. "10% of this, 15% of that"). Other looks referenced: Cullen Kelly's Pro Pack as a finishing LUT, Nova/Helios/Soul/Bloom/Dusk from his own pack.

## Teaching points
- The whole craft reduces to TWO buckets: Corrections (exposure, balance, contrast — photometric, 'could've been done in camera,' preserve how the camera saw the world) and Look Development (saturation, hue, split-tone, LUTs — push past reality to create FEELING). Sorting every tool into one bucket demystifies the process.
- WORK IN A WIDE-GAMUT SANDWICH. Don't transform straight to Rec.709 and grade there — that compresses your colors. Put an input CST (camera → DaVinci Wide Gamut) and an output CST (DWG → Rec.709 2.2) as bookends, grade everything in between inside DWG (wider even than RED Wide Gamut). Beginners 'just cranked contrast and maxed saturation' on raw log — the wrong way.
- A color space is really just RGB channel INTENSITY, and gamma is the intensity of the 'light behind the stained glass.' You could technically do a transform by hand with the RGB Mixer; the CST just automates it. Understanding this removes the 'magic plugin' mystique.
- SET TETRAHEDRAL 3D-LUT INTERPOLATION. The default Trilinear causes banding and crushed/compressed LUT results. This single project setting is a constant gotcha he forgets and regrets.
- GRADE UNDER YOUR DELIVERY. Output Rec.709 GAMMA 2.2 (not 2.4) for MacBook/iPhone/YouTube; use a non-baked monitor viewing LUT so what you see is what renders. 2.4 is still right for cinema/broadcast.
- EXPOSURE = HDR GLOBAL WHEEL, NEVER OFFSET. Offset lifts uniformly and clips data off the top; the HDR Global wheel bends the curve, anchors the black point, and rolls off highlights softly. Exposure is your broadest brush.
- WHITE BALANCE = GAMMA-LINEAR + GAIN WHEEL ONLY, never Temp/Tint. Setting the node to linear gamma and nudging only Gain balances without compressing/destroying color data the way the Temp slider does (credited to Cullen Kelly).
- JUDGE CONTRAST WITH SATURATION AT ZERO — color deceives the eye about tonal separation. Contrast is just black-to-white separation around a PIVOT.
- PIVOT = MIDDLE GRAY, and middle gray is NOT the slider center. In DWG it sits ~0.336 (find it with a mono middle-gray DCTL). Lock contrast to it so the image expands around a fixed fulcrum; push the pivot into sky/shadow to STEER the viewer's eye (the eye goes to the most intense point).
- Many Resolve tools do 'the same thing' but with different MATH: Saturation = multiplier (blows out already-saturated areas), Color Boost = additive/vibrance (invents saturation, breaks shadows), Color Slice Sat = deep/natural. Learn the one or two best versions and ignore the rest. He uses Color Slice Saturation.
- Use FREE DCTLs over clunky built-ins: a 3x3 Matrix DCTL (with Tetra interp) for skin correction beats the RGB Mixer because you don't have to manually rebalance every channel; a TetraInterpHSV DCTL beats Hue-vs-Hue (which bands) for granular hue/sat/value.
- SPLIT-TONING lives in the curves with LUMINANCE MIX = 0, anchored at middle gray — but you usually don't need to do it by hand. Film-emulation LUTs already encode tried-and-true, eye-pleasing split-tones; let the LUT do it and grade underneath it.
- Film LUTs (Kodak 2383 etc.) wash/lift the blacks. Either build/choose a LUT adapted to hold the low end (his 'Terra'), or restore shadow detail with correction nodes UNDERNEATH the LUT.
- The HK (Helmholtz-Kohlrausch) DCTL is a subtle perceptual finisher — the eye reads some saturated colors as brighter; HK evens that out. 'Icing on everything.'
- TEXTURE LAST and SPARINGLY: film grain belongs in shadows (pull it out of highlights/mids); Contrast Pop adds bite; a touch of blur emulates a pro-mist/black-satin diffusion. The best grades are invisible.
- PHILOSOPHY OVER TECHNIQUE: 'do the most with the least.' A few well-understood correction nodes + a good LUT beat a 30-node tree. Approach matters more than technical horsepower; there are no wrong answers, just more/less ideal ways.
