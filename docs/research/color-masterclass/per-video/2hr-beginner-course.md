# 2hr-beginner-course — analysis

*Source: Ragib Choudhury, 159K views · framesRead: 24*

## Overview
The entire method is a reusable node-tree workflow split into three conceptual stages: (1) BALANCE the log/flat image into a neutral, full-range picture; (2) ISOLATE color via a layer-mixer (skin / second object / background) so each region can be steered independently; (3) design the LOOK on a parallel-mixer block of global+adjustment nodes, then finish with a serial fine-tune, film grain and sharpening. The governing philosophy stated repeatedly: 'lots of small changes make a big change' — never push any single tool (saturation, color separation, skin shift) hard, because flat or low-bit footage breaks up. Color management is set first (DaVinci Wide Gamut / Intermediate timeline; the powergrade's CST node converts each camera's log into that space, and a CST-out node converts to Rec.709 for delivery). Skin lives in the midtones (gamma) and is checked against the vectorscope skin-tone indicator line. The signature look taught is a desaturated cool blue/teal with warm skin — 'not bleach bypass but very desaturated' — plus optional grain and a glow for dreamier shots.

## Order of operations
1. 0. SET COLOR MANAGEMENT FIRST (Edit tab > Project Settings > Color Management): Timeline color space = DaVinci Wide Gamut / DaVinci Intermediate; Output = Rec.709-A on Mac, or Rec.709 Gamma 2.2/2.4 on Windows, so the export matches what you see.
2. 1. APPLY THE POWERGRADE: right-click the starter powergrade still in the Gallery > Apply Grade. This drops the full ~21-node tree onto the clip.
3. 2. SET THE CST INPUT: in the 'CST in' node enter your camera's color space + gamma (e.g. S-Gamut3.Cine / S-Log3 for Sony; Canon Cinema Gamut / Canon Log 3; Blackmagic Gen 5 Film; Rec.709/Rec.709 for iPhone non-log). This neutralizes the log curve.
4. 3. PICK A HERO FRAME: scrub to one representative, well-exposed frame and grade from there.
5. 4. BALANCE (yellow nodes: Exp/bal, Contrast, Sat). Stretch the waveform/parade to fill 0-1023 without clipping highlights (1023) or crushing blacks (0). Use Lift=shadows, Gamma=midtones, Gain=highlights, Offset=global; add contrast and shift the Pivot left toward shadows if muddy; add a little saturation. Fix white balance here with Temp/Tint if needed.
6. 5. ISOLATE COLOR (layer-mixer: Main object on the BOTTOM layer = top priority, then 2nd object, then Background). Qualify the skin on the Main-object node, refine with Hue/Sat/Lum width + softness + denoise/blur, optionally back it up with a Power Window + tracker, pen tool, or Magic Mask. Repeat for a 2nd object (jacket/sky/grass). The background node then receives a subtle tint that does NOT touch the isolated regions.
7. 6. CHECK SKIN: parade > Vectorscope, enable Show Skin Tone Indicator; nudge skin (on gamma) onto the skin line; don't over-push at this stage.
8. 7. DESIGN THE LOOK (parallel-mixer block fed by one Global node, plus ADJ 1-3 and PW 1-3). On Global use curves (3 points: lift shadows-contrast, raise mid, lower highs) + primaries to set tonality and color cast (e.g. blues into gain, oranges into highlights/skin). Use Power Windows for vignette/sky/subject brightening. Use Adjustments for color slicing (HSV sat + density), color warper, HSL curves, face refinement.
9. 8. FINAL LOOK (serial node after Global): push the whole image up/down, desaturate, add final color cast — this is where the strong creative move goes because it works on the combined global result.
10. 9. FINE TUNE (serial): tiny white-balance/contrast tweak; optional gaussian blur to soften for a filmic feel.
11. 10. FILM GRAIN node: Studio Film Grain OFX (e.g. 16mm/35mm) or the free overlay clip set to Overlay/Cmd-F composite. Adjust strength/size/softness; can limit grain to mids/highlights via Advanced.
12. 11. SHARPEN last (after grain): blur tool radius below 0.50 to sharpen (he uses ~0.46-0.48), and raise 'scaling' to recover micro-detail for social compression.
13. 12. OPTIONAL output blanking (2.39:1 or 1.33:1) for cinematic crop.
14. 13. DELIVER tab: Custom Export, MP4 + H.264, set in/out points (I/O), timeline resolution/frame rate, quality Automatic, hardware acceleration on; Add to Render Queue > Render. Turn grain back on only for final export if playback is slow.

## Node tree
The 'DaVinci Resolve Starter Powergrade' template (~21 nodes), read off frames f004/f006/f014/f030: [01 CST in] -> [02 Exp/bal] -> [03 Contrast] -> [04 Sat] (the four yellow BALANCE nodes in serial) -> into a LAYER MIXER feeding three isolation nodes: [05 Background] (top layer), [06 2nd obj], [07 Main obj] (bottom layer = highest priority). Layer-mixer output -> [09 Global] which fans out through a PARALLEL MIXER to the LOOK block: [10 ADJ 1] [11 ADJ 2] [12 ADJ 3] and [13 PW 1] [14 PW 2] [15 PW 3] (three adjustment + three power-window nodes, all reading from Global in parallel). Parallel output recombines -> [17 Final look] -> [18 Fine Tune] -> [19 Film grain] (added during the course) -> [CST out] (DaVinci WG/Int -> Rec.709) -> [20/21 Sharpening]. Nodes are labeled via F2 (must bind 'Label selection node' in Keyboard Customization). New serial node = Alt/Opt-S; new layer of isolation = Alt/Opt-L; toggle a node = Cmd-D; before/after of a node group = select + Cmd-D, or Shift-D to hide all grades.

## Scopes
- **Parade (RGB) — his default for balancing** — Used while moving Lift/Gamma/Gain/Offset to confirm you have full tonal information across the image before any color isolation; a flat log image sits as a thin band in the middle. · targets: Stretch the three channels to fill the full vertical range: top = 1023 (highlights), middle = midtones, bottom near 0 (shadows). Do NOT let highlights hit/flatten against the 1023 line (clipping) or blacks flatten at 0. Scale shown 0-1023 (10-bit).
- **Waveform** — Offered as the common alternative graph; he prefers parade but says many colorists use waveform. Drag the panel divider to enlarge for clarity. · targets: Same goal as parade — spread the luma; highlights toward top, shadows toward bottom without clipping.
- **Vectorscope + Show Skin Tone Indicator** — Enabled via the scope's three-dots menu > Show Skin Tone Indicator. While adjusting skin on gamma, watch the cluster ride onto the line. Don't over-push during isolation; revisit during the look. · targets: Skin should fall ON the skin-tone line (the indicator drawn between red/yellow, the classic 'I-line'). Off the line = unrealistic skin unless stylistic.
- **Histogram / CIE** — Mentioned as options in the Scopes panel menu. · targets: Available in the scope dropdown (Parade / Waveform / Vectorscope / Histogram / CIE Chromaticity) but not used as primary in this course.

## Primaries
Primaries = Color Wheels in DaVinci-wheel mode. Mapping taught explicitly: LIFT = shadows, GAMMA = midtones (where skin lives), GAIN = highlights, OFFSET = global/overall look. Master sliders under each wheel plus Contrast and Pivot at top. Balancing workflow: Offset to set overall exposure, Gamma up to open midtones, Lift down to set blacks, Gain to place highlights — watching the parade so nothing clips. Contrast added with Pivot pushed LEFT (toward ~0.435 seen on screen, default ~0.5) so contrast bites shadows not highlights when the image is muddy. White balance handled with Temp/Tint at the top of the wheels panel. For the look, color is injected by tinting specific ranges — e.g. blues into Gain (highlights) and oranges/warmth into Gamma (midtones/skin), creating a split-tone. On-screen Gain RGB values during the look example read R1.00 G0.91 B1.02 / 1.04 (cool gain). Studio tip: use HDR Wheels instead of primaries for exposure balance when available — far more control over recovering clouds/highlights (shown bringing detail back into a bright sky).

## Secondaries
Qualifier (HSL) is the primary isolation tool: enable qualify-plus, click the skin, press Shift-H to view the selection (or the half-circle highlight button). Refine with Hue Center/Width, Saturation Low/High + L.Soft, Luminance Low/High + softness, plus Matte Finesse Denoise + Blur Radius (he keeps both under ~20). On-screen skin-qualifier values observed: Hue Center 31.8 Width 14.7 Soft 2.8; Sat Low 0.5 High 7.3; Lum Low 51.9 High 64.7 L.Soft 2.6 H.Soft 2.6 (frame f012). Backup isolation methods, in order: (1) Power Windows — circle/radial, square, gradient, plus the PEN tool to draw around a subject, with Softness feathering and 'Invert' (key output) for vignettes; (2) Tracker — press forward/back to track the window across motion; (3) Magic Mask (Studio only) — draw a stroke on the subject, track forward/back for an almost-perfect auto matte, then still refine in the qualifier. Color slicing (ColorSlice, Resolve 19+) is used as an HSV move: its Saturation raises saturation WITHOUT lifting luminance/contrast the way the normal sat slider does; Density fills the image subtly; per-color (Red/Skin/Yellow/Green/Cyan/Blue/Magenta) Hue/Sat control. Color Warper (6- or 8-sided hexagon) shifts hue/sat of a clicked color; raise its radius if the selection is rough. HSL curves (Hue vs Hue, Hue vs Sat, Hue vs Lum, Lum vs Sat, Sat vs Sat, Sat vs Lum) for targeted tweaks — e.g. Sat vs Sat to pull down the most-saturated yellows. Glow OFX (Shine Threshold sets which brightness glows; Spread for bloom; Color Filter to tint the glow) for dreamy skies.

## Skin tone
Skin is treated as a midtone phenomenon: 'skin normally falls into the midtones and midtones is correlated with gamma.' The check is the Vectorscope with Show Skin Tone Indicator enabled (three-dots menu); the skin cluster should sit on the skin-tone line. He nudges skin placement on the Gamma wheel (or via Hue vs Hue) and confirms against the line. Critically, because skin is ISOLATED on the Main-object node, after any global/cool look shift you return to that node and re-place or desaturate the skin so it stays warm and natural against the cool background ('the beautiful thing about color isolation is we've already isolated it, so we can come in and adjust it'). For Sony skin he shifts hue slightly toward orange off the reds; he warns against pushing skin saturation too 'hot'. Optional Face Refinement OFX (detect face > track) further controls skin: smoothness (lower the 'scaling' inside it to keep texture, not plasticky), skin-tone color boost, shine removal, plus eyes brighten / eye-light / eye-bag removal, lips/teeth/blush — 'many small adjustments create a huge change.'

## Color management
Set BEFORE grading, in Project/Master Settings > Color Management. Timeline color space = DaVinci Wide Gamut / DaVinci Intermediate (DWG/Int), not the default Rec.709 Scene. Output color space = Rec.709-A on Mac (makes the export match the color-tab image exactly), or Rec.709 Gamma 2.2 / 2.4 on Windows. Inside the powergrade, the actual transforms are done by nodes, not the project's automatic CM: the 'CST in' node (Color Space Transform OFX) takes the camera's native log (Sony S-Gamut3.Cine/S-Log3, Canon Cinema Gamut/Canon Log 3, Blackmagic Gen 5 Film, or Rec.709/Rec.709 for iPhone) and converts INTO DaVinci WG/Int as the working space; the 'CST out' node near the end converts from DWG/Int back to Rec.709 for delivery. For non-log iPhone footage you set CST in to Rec.709 in / Rec.709 gamma — which technically does 'nothing' tonally but puts the clip into the same node pipeline so the rest of the tree works identically.

## Look design
Primary look = desaturated COOL TEAL/BLUE with preserved WARM SKIN ('not bleach bypass but very desaturated; blue and orangey'). Built on the Global node: a 3-point tone curve (lift+contrast the shadows, gently raise mids so skin opens up, pull highlights down), then blues pushed into Gain/highlights and the image darkened via the gain black slider (he 'prefers darker imagery'), overall saturation lowered, and oranges added to highlights/skin to pull warmth back. Power Windows then sculpt: a gradient PW darkens the brighter side, an inverted circular PW makes a vignette, and a center PW lifts the subject's shadows to keep the face from being eaten by shadow. Adjustment nodes add color slicing (HSV sat + density) and color-warper hue nudges. The FINAL LOOK serial node makes the decisive push — drag the whole image down and desaturate further — followed by a Fine-Tune node for a last white-balance/contrast nudge and optionally a gaussian blur to emulate the softer look of 16/35mm film, then grain and sharpening. Alternate looks demonstrated: warm split-tone (blue shadows via Lift + orange mids via Gamma) on iPhone Faroe-Islands footage with a tinted glow on the sky; a green 'grungy/Joker-ish' grade on an underexposed S-Log3 clip done with barely more than a temperature shift + green offset; and a stylized pink-grass example to show qualifier-driven hue replacement. Output blanking (2.39:1 letterbox or 1.33:1) optionally added for cinematic framing.

## Teaching points
- 'Lots of small changes make a big change' — the core mantra; subtle multi-node moves beat any single hard push.
- Balance BEFORE you isolate: you can only get clean qualifier selections once the image has tonal/saturation separation. On flat log, selecting skin selects almost everything because all tones collapse together.
- Never push color separation or saturation too hard — flat and 8-bit/low-bit footage 'breaks up' and looks bad. iPhone/8-bit especially: the more saturated, the more flat/broken it looks; sometimes remove sharpening entirely since phones already sharpen.
- Bit depth determines selection quality: 8-bit struggles to qualify cleanly; 10/12-bit/raw isolate far better.
- The layer mixer enforces priority — the BOTTOM layer (Main object) wins over everything above it; turn nodes on/off (Cmd-D) to see exactly what each region selection is grabbing.
- Parallel mixer for the global look means all look-nodes read from one Global node, so you don't lose information the way stacked serial nodes do; use serial only afterward when you intentionally want to build on the combined result.
- Skin lives in the midtones/gamma; verify with the vectorscope skin-tone line; because skin is isolated you can always come back and re-place it after any global shift.
- Use a reference still (import into Gallery, split-screen, watch its parade) to copy tonality when unsure where to take a grade; study film stills and social inspiration.
- Add film grain for texture/interest; clean is fine but grain adds life. Grain is GPU-heavy — work with it off (Shift-D hides grades) and re-enable only for final export, or drop timeline to 1080p / lower playback speed.
- Sharpen with the blur tool radius BELOW 0.50 (lower = sharper); add sharpening AFTER grain so it adds extra texture; the rarely-used 'scaling' slider recovers striking micro-detail, great for social-media compression.
- Grades don't have to be realistic — color is expression; if a stylized/unrealistic look serves the story, go for it (his own Instagram grades are intentionally 'funky').
- Sometimes a great grade is just a CST + a little balancing + a single primary-wheel shift; you don't always need the whole tree.
