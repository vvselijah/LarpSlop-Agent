## 4. Architecture / anti-rot upgrades

Agent-practice improvements for the hub itself (from agent-practices sweep + the activation finding).

1. **Close the measure/generate seam on every new engine.** The 7-engine activation gap exists because "build" and "schedule" were never separated. Standing pattern: a built engine isn't "done" until it's either in an automated bat OR explicitly documented as on-demand-only. Add this to the dev-workflow Ship gate.
2. **ACE Reflector/Curator on memory (anti-collapse).** `self-improve/grade.py` is already a proto-Reflector; pair it with a quarterly `consolidate-memory` Curator pass so `team/memory.md` reconciles contradictions instead of growing monotonically (ACE, ICLR 2026).
3. **Honest context ceiling.** Replace the inflated %-hook trigger with a token-count trigger (~300–400k effective on the 1M window) in the 40%-rule note — the hook reads ~5× high (Chroma Context Rot, 2025; corroborated 2026).
4. **Verify-the-field discipline as a rule.** The `share`→`sends` and `reels_skip_rate`/`reposts` items both hinge on API fields not yet confirmed exposed. Make "verify the named API field exists before a plan promises it" an explicit dev-workflow review-lens question.
